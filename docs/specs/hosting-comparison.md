# WordPress ホスティング比較

> Codex セカンドオピニオン (2026-06-16) で「Cloud Run + Cloud SQL 自前運用は要件に対して重い」と指摘。マネージド WP を含めて運用観点で比較する。

## 評価軸

「月額費用」だけで判断せず、以下の運用観点を重視する:

| 軸 | 内容 |
|---|---|
| 障害対応 | 障害時のサポート対応速度・SLA |
| WP コア / プラグイン更新 | 自動更新可否、更新失敗時のロールバック |
| WAF | Web Application Firewall の標準提供有無 |
| バックアップ | 頻度・保持期間・復元手順の容易さ |
| 監視 | アクセス / エラー / リソース監視の標準提供 |
| 復旧時間 | 障害発生から復旧までの目安時間 (MTTR) |
| 担当者不在運用 | 本田氏不在時に他者でも対応できるか |
| ismap 適用要否 | 個人情報棚卸し結果次第で必要 / 不要が変わる |
| 月額費用 | 参考まで |

## 候補

### 1. Cloud Run + Cloud SQL (GCP 自前運用)

| 軸 | 評価 |
|---|---|
| 障害対応 | GCP サポート (Standard プラン以上で営業時間 4h SLA) |
| WP 更新 | 自前で WP-CLI / コンテナ rebuild → 失敗時は Cloud Run revision ロールバック |
| WAF | Cloud Armor (別契約) |
| バックアップ | Cloud SQL 自動バックアップ (7 日) + on-demand |
| 監視 | Cloud Logging + Monitoring (標準) |
| 復旧時間 | 自前対応のため担当者次第 |
| 担当者不在運用 | **△ 自前運用前提のため属人化リスク** |
| ismap 適用要否 | ismap 対応 GCP リージョンを選べば適合 |
| 月額費用 (目安) | Cloud Run: ¥1k〜 / Cloud SQL db-g1-small: ¥3-4k / Cloud Armor: ¥5k / 合計 ¥10-15k |
| ✅ 利点 | GCP 内完結 (ismap)、コンテナ運用ノウハウ流用 (aozora-sns-auto)、コスト柔軟 |
| ⚠️ 欠点 | 運用責任が全部自社、属人化、WP 更新オペレーション工数 |

### 2. Kinsta

| 軸 | 評価 |
|---|---|
| 障害対応 | 24/7 チャットサポート (英語、日本語対応あり) |
| WP 更新 | コアは自動、プラグインは管理画面から、ステージング切替容易 |
| WAF | Cloudflare Enterprise + DDoS 対策 標準 |
| バックアップ | 自動日次バックアップ 14 日 (プラン依存)、ボタン 1 つで復元 |
| 監視 | リソース監視ダッシュボード標準、APM 提供 |
| 復旧時間 | MTTR 30 分〜数時間 (サポート介入時) |
| 担当者不在運用 | **◎ サポートに連絡で大半対応可能** |
| ismap 適用要否 | **インフラは Kinsta 管理 (Google Cloud Platform 上だが Kinsta が運用)、ismap 適用不可** |
| 月額費用 (目安) | Starter $35/月 (約 ¥5,500) / Pro $70 (¥11,000) |
| ✅ 利点 | 運用負荷小、サポート手厚い、復旧速い、GCP 上で動く (内部的に) |
| ⚠️ 欠点 | ismap 求人サイトに必要なら不採用、英語サポート慣れが必要 |

### 3. WP Engine

| 軸 | 評価 |
|---|---|
| 障害対応 | 24/7 チャット / 電話 (英語) |
| WP 更新 | コア / セキュリティパッチ自動、メジャー更新は手動オプション |
| WAF | Global Edge Security (Cloudflare Enterprise) |
| バックアップ | 自動日次 30 日、ワンクリック復元 |
| 監視 | Application Performance Monitoring 標準 |
| 復旧時間 | MTTR 30 分〜数時間 |
| 担当者不在運用 | **◎** |
| ismap 適用要否 | Kinsta 同様、ismap 適用不可 (WP Engine 管理) |
| 月額費用 (目安) | Startup $30/月 (約 ¥4,700) |
| ✅ 利点 | 大手 WP 専業ホスト、信頼性高 |
| ⚠️ 欠点 | ismap 要件下では不採用、英語前提 |

### 4. エックスサーバー (国内)

| 軸 | 評価 |
|---|---|
| 障害対応 | 24/7 メール / 平日電話 (日本語) |
| WP 更新 | WP 自動更新標準、簡単インストール / バックアップ機能 |
| WAF | 標準提供 (シグネチャ更新は自動) |
| バックアップ | 自動 14 日 |
| 監視 | 管理画面で基本情報 |
| 復旧時間 | MTTR 数時間 |
| 担当者不在運用 | **◯ 日本語サポートで対応可能** |
| ismap 適用要否 | **× エックスサーバーは ismap 対応外** |
| 月額費用 (目安) | スタンダード ¥1,100/月 |
| ✅ 利点 | 安い、日本語サポート、国内ホスト |
| ⚠️ 欠点 | ismap 要件下では不採用、機能面で Kinsta 等に劣る |

### 5. さくらインターネット マネージド (RS-Lite / マネージドサーバ)

| 軸 | 評価 |
|---|---|
| 障害対応 | 平日電話 / メール (日本語) |
| WP 更新 | 手動 |
| WAF | オプション (別料金) |
| バックアップ | 自動・手動 (プランによる) |
| 監視 | 基本 |
| 復旧時間 | 数時間 |
| 担当者不在運用 | **△ サポート時間制限あり** |
| ismap 適用要否 | **× さくらインターネットは ismap 対応外** |
| 月額費用 (目安) | RS-Lite ¥800〜 |
| ✅ 利点 | 国内最安、長年実績 |
| ⚠️ 欠点 | ismap 不適合、機能面で限定的 |

### 6. Amimoto (国内 / WordPress 特化、AWS 上)

| 軸 | 評価 |
|---|---|
| 障害対応 | 営業時間 (日本語) |
| WP 更新 | 半自動 |
| WAF | あり |
| バックアップ | あり |
| 監視 | あり |
| 復旧時間 | 数時間 |
| 担当者不在運用 | **◯** |
| ismap 適用要否 | **AWS 上で動くため AWS の ismap 適合範囲 (東京・大阪) を選べば可** |
| 月額費用 (目安) | 数万円 (プラン依存) |
| ✅ 利点 | WP 特化 + 日本語 + ismap 適合 (条件付き) |
| ⚠️ 欠点 | コスト中、AWS 採用となる (GCP 統一方針と乖離) |

## 判断フロー

```
個人情報棚卸し (docs/specs/acceptance-criteria.md F-1, F-2)
  ↓
[ ismap 適用が必須か? ]
  ├ Yes → [1] Cloud Run + Cloud SQL (GCP) または [6] Amimoto (AWS)
  │        ├ GCP 統一 / 自前運用 OK → [1]
  │        └ WP 特化サービス / 日本語サポート希望 → [6]
  └ No  → [2] Kinsta / [3] WP Engine / [4] エックスサーバー
           ├ 運用負荷最小化 + コスト中 → [2] or [3]
           └ コスト最優先 → [4]
```

## 推奨ライン

**個人情報棚卸し結果次第で 2 系統:**

- **ismap 必須の場合**: **[1] Cloud Run + Cloud SQL** (aozora-sns-auto のノウハウ流用、Terraform で IaC 化、月額 ¥10-15k)
  - 緩和策: 運用手順書を徹底文書化、復旧訓練を四半期 1 回、属人化リスク軽減
- **ismap 不要の場合**: **[2] Kinsta** (運用負荷最小、サポート手厚い、月額 ¥5,500〜)
  - 緩和策: 英語サポート慣れ、必要なら日本語ドキュメント / 翻訳
