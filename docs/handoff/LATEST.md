### 技術的知見(新規)

- **国土地理院AddressSearch API**(`https://msearch.gsi.go.jp/address-search/AddressSearch?q=<住所>`)は番地レベル住所から`[lng,lat]`を無料・キー不要で取得可能。13拠点のジオコーディングに使用、結果は`build_geo_data.py`にハードコード(実行時ネットワーク依存を避けるため)
- **CSS `aspect-ratio` + 片方向paddingの落とし穴**: `aspect-ratio`はborder-box(padding込み)に適用されるため、`padding: Npx 0`(縦のみ)のような非対称paddingを持つ要素にaspect-ratioを設定すると、content-box(実際に子要素が使える領域)の縦横比が指定値からズレる。子要素(SVG等)がpreserveAspectRatioでレターボックスされ、%指定の絶対配置がズレる。padding は aspect-ratio 制約のある要素ではなく親要素側に持たせること
- **非同期fetchで生成するDOM要素と、生成前に発生し得るユーザー操作の状態同期**: `fetch().then(html => el.innerHTML = html)`のようなパターンで動的にインタラクティブ要素(ピン等)を生成する場合、生成が完了する前にユーザーがフィルタやボタン操作を行うと、その時点で対象要素が存在せず状態が反映されないまま取り残される。生成直後に現在の状態を明示的に再同期する処理(本セッションでは`applyNearestPinHighlight()`+`applyFilters()`の再呼び出し)が必要
- **`geolonia/japanese-prefectures`**(GitHub、GFDL、Wikipedia日本地図.svgベース)は都道府県ごとに`<g data-code="NN">`でグループ化された軽量(30KB)SVG地図で、検索UI用途を想定した設計(`class`でCSS色分け、`data-code`でJS操作)。九州7県のみ抽出する際、鹿児島県は奄美・種子島・屋久島等の遠方離島群が同一グループ内に別ポリゴンとして混在するため、ローカル座標のx値で本土+近隣島嶼のみをフィルタする必要があった(実データ確認要)
