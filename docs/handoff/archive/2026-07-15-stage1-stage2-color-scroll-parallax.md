### 技術的知見 (新規)

- **`background-size:cover` は要素自身のボックスの縦横比で幅基準/高さ基準が決まり、閾値は画像のアスペクト比・要素の実寸に依存して動的に変わる**。固定の viewport 幅ブレークポイントでは安全性を保証できない(1200-1390px帯で実害バグが再発した実例あり)。視差スクロール等でボックスを一時的に拡張する実装をする際は、`Image()` で自然サイズを実測し、拡張後も基準軸が変わらないことを確認してから有効化する必要がある。かつ要素の高さが responsive (`clamp()` 等) な場合、`resize` イベントでも再判定が必要
- **`tokens.css` のグローバル `h1,h2,h3,h4 { color: var(--color-heading) }` ルールは、色付き背景カード内に配置された見出し要素の親由来の文字色継承を直接上書きする**。カード内見出しには明示的に `color: inherit` が必要 (career-ladder__role で発覚、以降同様のパターンに注意)
- **ローカル `python3 -m http.server` はデフォルトで `Content-Type` に `charset=utf-8` を付与しない**ため、日本語コメント入りCSSでmojibake→カスタムプロパティ消失が起きうる(既知、`reference_python_http_server_charset_mojibake.md` 参照)。またブラウザのHTTPキャッシュがCSS更新を追従しないことがあり、`Cache-Control: no-store` ヘッダを返すカスタムサーバーでも既存キャッシュエントリには遡及しないため、疑わしい場合は `<link>` の href にキャッシュバスティングクエリを付与して確認するのが確実
- **`html { scroll-behavior: smooth }` が設定されている場合、`window.scrollTo(x, y)` の2引数形式も smooth スクロールとして扱われる**(legacy overload は `{behavior:'auto'}` 相当としてCSSのscroll-behaviorを尊重する仕様)。Playwright等でスクロール位置を即座に確定させたい場合は `scrollTo({top, left, behavior: 'instant'})` を明示する必要がある
